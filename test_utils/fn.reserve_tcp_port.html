pub fn reserve_tcp_port() -> Result<u16>
Expand description

Reserve a TCP port using the TIME_WAIT trick.

This function allocates an ephemeral port and forces it into TCP TIME_WAIT state by completing a TCP handshake. The port is then:

  • Protected from OS ephemeral allocation for ~60 seconds (TIME_WAIT duration)
  • Still available for explicit binds (TIME_WAIT prevents collision with ephemeral allocation, not intentional rebinding)

This pattern is based on Yelp’s ephemeral-port-reserve approach and is useful when you need to know a port number before starting a service, but the service needs to bind to the port itself (not use a pre-bound listener).

§Errors

Returns an error if unable to bind, connect, or accept on 127.0.0.1.

§Example

use test_utils::reserve_tcp_port;

let port = reserve_tcp_port().expect("OS should have ephemeral ports available");
// Port is now in TIME_WAIT state - protected from ephemeral allocation
// Services can still bind to it explicitly
